<Tabs>
  <Tab title="Precompiled binaries">
    The fastest way to get started is to download a precompiled binary from the [GitHub Releases page](https://github.com/codedeliveryservice/Reckless/releases).

    Each release includes three binary variants optimized for different CPU capabilities:

    | Variant | Description |
    | ------- | ----------- |
    | `-avx512` | Fastest. Requires a recent CPU with AVX-512 support. |
    | `-avx2` | Fast. Supported on most modern CPUs (post-2013). |
    | `-generic` | Compatible with virtually all CPUs, but significantly slower than the AVX2 or AVX-512 builds. |

    <Note>
      If you are unsure which binary to use, try the AVX-512 build first. If it doesn't run on your system, fall back to the AVX2 build, or the generic build as a last resort.
    </Note>

    Once you have downloaded the binary, make it executable (on Linux and macOS) and you are ready to connect it to a GUI.

    ```bash
    chmod +x reckless-avx2
    ```
  </Tab>

  <Tab title="Build from source">
    Building from source lets you compile Reckless optimized specifically for your CPU. This is the recommended approach if you want the best possible performance or need to customize the build.

    ### Prerequisites

    Before building, make sure you have the following installed:

    - **Rust 1.88.0 or later** — install via [rustup](https://www.rust-lang.org/tools/install)
    - **Clang** — required to compile the [Fathom](https://github.com/jdart1/Fathom) library used for Syzygy endgame tablebase support

    ### Standard build

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/codedeliveryservice/Reckless.git
        cd Reckless
        ```
      </Step>

      <Step title="Build with Cargo">
        Run the following command to compile a release binary optimized for your CPU:

        ```bash
        cargo rustc --release -- -C target-cpu=native
        ```

        The compiled binary is placed at `./target/release/reckless`.
      </Step>

      <Step title="Verify the build">
        ```bash
        ./target/release/reckless compiler
        ```

        This prints the compiler version, target architecture, and feature flags used for the build.
      </Step>
    </Steps>

    ### Build without Syzygy support

    If you do not have Clang installed or do not need Syzygy endgame tablebase support, add `--no-default-features` to skip the Fathom dependency:

    ```bash
    cargo rustc --release --no-default-features -- -C target-cpu=native
    ```

    ### PGO build

    Profile-guided optimization (PGO) uses runtime profiling data to produce a faster binary. PGO builds typically outperform standard release builds.

    <Steps>
      <Step title="Install the required tools">
        ```bash
        rustup component add llvm-tools
        cargo install cargo-pgo
        ```
      </Step>

      <Step title="Run the PGO build">
        ```bash
        make pgo
        ```

        This instruments the binary, runs the built-in benchmark to collect profiling data, and then recompiles with those profiles applied. The final binary is placed at `./reckless`.
      </Step>
    </Steps>

    You can also run the PGO steps manually if you don't have `make` available:

    ```bash
    cargo pgo instrument
    cargo pgo run -- bench
    cargo pgo optimize
    # Binary: ./target/<target-tuple>/release/reckless
    ```
  </Tab>
